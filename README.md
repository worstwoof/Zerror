# 📖 项目简介 

传统错题本更多承担“记录错误”的功能，却很少真正帮助用户**理解错误、消化错误、转化错误**。对于许多学生来说，错题积累得越多，焦虑感反而越强，复习过程也容易陷入机械重复、缺少反馈和动力的泥潭。

**知芽 Zerror** 以“**AI 错题重构**”为核心思路，尝试将错题整理从静态记录升级为动态成长系统。
项目围绕“**识别 - 诊断 - 重构 - 训练 - 复习**”的闭环展开，通过多模态 AI 能力将错题转化为更有记忆点、更具陪伴感的学习内容，让复习体验从“回避错误”转向“理解错误、利用错误、超越错误”。

## ✨ 核心功能亮点

### 1. 📸 智能错题收录
- **多渠道导入**：支持拍照上传、图片导入与手动录入。
- **OCR 减负**：结合图像识别能力提取题面内容，极大降低人工整理成本。
- **复杂内容支持**：支持数学公式、理科符号等复杂内容的后续扩展录入。

### 2. 🧠 知芽 AI 深度解析
- **结构化诊断**：基于大语言模型，自动提炼知识点标签、难度信息与关键考点。
- **降维解构**：输出更适合学生理解的解题思路、步骤拆解与提示总结。
- **陪伴式反馈**：生成带有生命感与陪伴感的“知芽 AI 解析”，有效降低用户对错题的抗拒与焦虑心理。

### 3. 🔄 举一反三训练生成
- **逻辑迁移**：在原题基础上提取底层逻辑，一键生成相似题或变式题。
- **定向爆破**：通过“同知识点、不同问法”的训练，帮助用户形成知识迁移能力，攻克薄弱点。

### 4. 💎 错题资产可视化沉淀
- **告别死板列表**：将错题结果组织为具有辨识度的学习资产，而非枯燥的题目堆积。
- **视觉化载体**：引入“知植闪卡”、“知识精灵”等视觉化表现形式，增强记忆性。
- **粘性提升**：强化收集感与陪伴感，大幅提升用户的持续复习意愿。

### 5. 🌱 复习闭环与成长体验
- **全链路整合**：将错题整理、知识理解、重复训练与阶段复盘无缝衔接。
- **知其所以然**：不仅提示“错了”，更引导“为什么错、以后如何避免”。

## 💡 设计解读与创新评估 

### 一、 理念贯穿性
项目的核心理念是：**“不在错误中焦虑，让知识在灵感中发芽。”**
- **命名哲思**：“知芽”象征知识萌发；“Zerror” 则象征着 “zero error 零失误” 的意义，视其为生长的起点。“知芽” 与 “Zerror” 两者读音巧妙相近，相得益彰。
- **情绪价值**：摒弃传统学习工具的严肃压迫感，界面采用具备呼吸感和自然感的暗调卡片式风格，打造“养成系”学习体验。

### 二、 核心创新点
1. **升维“错题本”概念**：从“静态存储工具”升级为“动态 AI 成长系统”。
2. **情绪与认知双管齐下**：在提供高效率学习方案的同时，通过友好的交互文案抚平面对错误时的挫败感。
3. **多模态学习体验**：融合图像输入、LLM 文本生成与 Stable Diffusion 视觉化内容构建，打破纯文本阅读疲劳。

### 三、 市场前景
错题整理是中学生及高校群体的长期高频刚需。随着 AIGC 技术的普及，用户对“个性化辅导”与“智能化衍生练习”的需求激增。**知芽 Zerror** 结合了效率工具的实用性与养成类产品的粘性，在校园学习场景中具备极强的落地价值与传播潜力。

## 🛠️ 技术架构与技术栈 

项目采用 **前端客户端 + 后端服务 + AI 引擎** 的分层解耦架构，保障高耗时生成任务不阻塞主业务链路。

* **📱 客户端 (Frontend)**: Flutter, Dart, Material Design, Android Native/MethodChannel
* **⚙️ 服务端 (Backend)**: FastAPI, Python, Pydantic, SQLAlchemy, Celery
* **🗄️ 数据与基建 (Infrastructure)**: PostgreSQL, Redis, Docker / Docker Compose
* **🤖 AI 引擎 (AI Engine)**: 
  * LLM API (逻辑推理、解析生成)
  * OCR Vision (图像文本提取)
  * Prompt Engineering
  * Stable Diffusion (视觉化/闪卡合成)
